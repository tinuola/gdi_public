Array Iterators

Array iterators loop (iterate) through an array and perform an action on each item in the array 

// Using .map() array iterator

// Square or multiply each number in the array by itself

let numberList = [1 ,2, 3, 4]

const doubledNumberList = numberList.map( num => {

  return num * num;

})

console.log(doubledNumberList) 

// Output: [1,4,9,16]

.map()

Loops through an array; performs an action on each element and returns a new array of the results 

// Add the string 'rock-' to each word in array

let wordList = ['bottom', 'candy', 'star', 'garden']

// Iteration:
let rockWordList = wordList.map( word => {

  return `rock-${word}`

})

console.log(rockWordList)

// Return new array:

// ['rock-bottom', 'rock-candy', 'rock-star', 'rock-garden']

Array.prototype.map() - MDN

.filter()

Loops through an array; checks if each item meets a specified condition; and returns a new array of the elements that pass the test 

// Filter out words that contain the string 'cat'
  
let arrayOfWords = ['avocado', 'advocate', 'catty', 'catacomb']

// Iteration:
let catWordList = arrayOfWords.filter( word => {

  return word.includes('cat')

})

console.log(catWordList) 

// Return new array: 

// ["advocate", "catty", "catacomb"]

How do array iterators work?

Array Iteration: Breakdown

An array iterator requires a second function to be passed as a parameter

This second function specifies what needs to be done to each element in the array

.map() Breakdown

// A function that adds the word 'rock-' to a string value

function addRockToWord(str){

  return `rock-${str}`

}
let wordList = ['bottom', 'candy', 'star', 'garden']

// .map() in its simplest form:

// let rockWordList = wordList.map()

// .map will accept addRockToWord as a parameter

let rockWordList = wordList.map(

  function addRockToWord(str){

    return `rock-${str}`

  }

)

console.log(rockWordList)

.filter() Breakdown

// A function that checks if a word contains the string 'cat'

function verifyWordHasCat(str){

  if(str.includes('cat')){
    return true
  }
  return false
}
let arrayOfWords = ['avocado', 'advocate', 'catty', 'catacomb']

// .filter() in its simplest form:

// let catWordList = arrayOfWords.filter()

// .filter will accept verifyWordHasCat as a parameter

let catWordList = arrayOfWords.filter(

  function verifyWordHasCat(str){

    if(str.includes('cat')){
      return true
    }
    return false
  }
)

console.log(catWordList)

What makes array iterator methods unique?

New Terms

Array iterators are types of functions known as higher order functions

A higher order function can use another function as a parameter or can return a function as an output

A function that can be passed as a parameter is known as a callback function

Array Iteration: Breakdown Syntax

// An array

let arrayName = [ item1, item2, item3, item4]

// An array iterator

arrayName.arrayIterationMethod( callbackFunction() {

  // Code to apply to each element

} )

Syntax Breakdown

In the previous examples, .map(), .filter(), the array iterators, are higher-order functions

addRockToWord, verifyWordHasCat--the functions passed as parameters to the iterator methods--are callback functions

Shorthand Syntax

The syntax we've used in the breakdown examples is considered verbose; you're more likely to see and use arrow functions with array iterators

Iterators and Arrow Function Syntax

Without arrow syntax

let rockWordList = wordList.map(
  function addRockToWord(str){
    return `rock-${str}`
} )

With arrow syntax

let rockWordList = wordList.map( (str) => {
  return `rock-${str}`
} )
  • Remove the function keyword and function name
  • Add an arrow before the first opening curly bracket

Iterators and Arrow Function Syntax

The rules we learned for writing arrow function shorthand apply with iterators

With only one parameter 'str' in the expression, we can omit the parentheses around it 

let rockWordList = wordList.map( str => {
  return `rock-${str}`
} )

Iterators and Arrow Function Syntax

If an arrow function expression is simple enough, we can omit the return keyword and keep the code to one line 

let rockWordList = wordList.map( str =>  `rock-${str}` )

Return Values

Some iteration methods create new arrays; others return numerical or boolean values

One particular iterator, forEach, does not return any value at all

.forEach()

Runs an action on every element in an array; unlike other methods, it does not return an actual value 

let ageList = [4,5,6,7]

const updatedAgeList = ageList.forEach( element => element + 5 ) 

console.log(updatedAgeList) // Returns undefined

.forEach() - MDN

.forEach()

The forEach behaves similarly to the for loop and the for...of loop 

let ageList = [4,5,6,7]


// .forEach()
ageList.forEach( element => console.log( element + 5 ) ) 


// for loop
for( let i = 0; i  < ageList.length; i++ ){

  console.log( ageList[i] + 5 )

}


// for ... of loop
for( let age of ageList ){

  console.log( age + 5 )

}


// Output: 9, 10, 11, 12
