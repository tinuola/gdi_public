A Gentle Introduction to ES6+

Variable Declaration

ES6 introduced two new keywords to declare variables

let and const

var Variable Keyword

  • The pre-ES6 way of declaring variables
  • Not recommended to declare variables this way anymore
  • But still used in legacy code
var yourName = 'Alina';

let Variable Keyword

  • Variables cannot be re-declared
  • Variables can be changed or re-assigned
let firstName;

let firstName = "Charmaine" // This will throw an error

let middleName = "Kenyatta"

middleName = "Michio" // Re-assigning the variable

let (MDN)

const Variable Keyword

  • Variables declared with const have to initialized
  • Variables cannot be re-assigned or re-declared
  • Used for values that should be constant or unchanged
const userName // Not initialized; this will throw an error

const userName = "Jasmine" // Initialized

userName = "Jackie" // Re-assigned; this will throw an error

const (MDN)

What else is unique about const and let?

Scope

Variables declared with const and let are block-scoped

Scope

Scope is how JavaScript manages the ability to use or access declared variables across a codebase

A Simple Explanation of Scope in JavaScript

Scope

There are three common levels of scope:

  • Global
  • Function
  • Block (Introduced in ES6)

A Simple Explanation of Scope in JavaScript

Global Scope

A variable declared in the global scope (i.e., not declared inside a function, a code block, etc.) can be used anywhere in the code 

let year = 2005; // Variable in global scope

// Used anywhere in the code
console.log (year)

// Used in a function
function printYear(){
  console.log(year)
}
printYear()
  
// Used in a block
if(true){
  console.log(year)
}

Function Scope

A variable declared inside a function can only be used in that same function 

function printNewYear(){

  let y2k = 2000;

  console.log(y2k);

}

printNewYear() // Output: 2000

console.log(y2k) // Error

Block Scope

A variable declared inside of a code block { } can only be used inside that block; for example, an if block 

let age = 20 

if(age === 20){
  
  let greeting = `Happy Birthday!`

  console.log(greeting) // Output: Happy Birthday!

}

console.log(greeting) // Error

Arrow Functions

Arrow Functions

Introduced in ES6, arrow functions provide an alternative and more concise way to write function syntax

An Introduction to JavaScript Arrow Functions

Arrow Functions

// Standard function syntax
function greeting(name){

  return `Hello ${name}`

}

// Using an arrow function expression
const greeting = (name) => {

  return `Hello ${name}`

}

console.log(greeting('Keisha'))

Arrow Function Syntax

const functionName = () => {

// Code to run

}

functionName()

const functionName = (parameter) => {

// Code to run

}

functionName(argument)

Arrow Function Shorthands

  • An arrow function with a single parameter can omit the parentheses around the parameter 
function greetFriend (firstName) {

  return `Hey ${firstName}`

}
  
// Arrow Function Shorthand

const greetFriend = firstName => {

  return `Hey ${firstName}`

}

Arrow Function Shorthands

  • An arrow function with a single or simple expression can omit the curly brackets and the return keyword 
function multiplyByFive (num) {

  return num * 5

}
  
// Arrow Function Shorthand

const multiplyByFive = num => num * 5

Caveat

If an arrow function has no parameter or has more than one parameter, the parentheses are required
