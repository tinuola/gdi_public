What is a Website?

What is a Website?

  • A single page or a group of connected pages that share a domain name
  • Typically provides static text and visual content; one-way information
  • Occasional user interactivity (Ex: Button clicks)

Website vs Web Application

Web Applications

  • Software or program that's accessible
    through a web browser
  • Websites with more complex functionality and development:
    • Authentication, Reactive/Interactive, State Management, eCommerce

Web Applications

  • NetFlix
  • Discord
  • Instagram
  • Facebook

Additional Terms to Know

  • Client-side or Frontend: Aspects of a website, web application or interface that a user sees or engages with
  • Server-side or Backend: Where data and source code is stored; where the "client-side" sends a request to and receives a response from

<head> Tag

Headings

Despite appearances, heading tags signify importance, not size

  • <h1> tags are used for main headings
  • <h2>, <h3> ..., all other tags are subheadings

Convention for Headings

  • There should only be one <h1> tag per page
  • All other tags can be used multiple times
  • For accessibility best practice, tags should be used in order:
    • Ex: An <h2> should come before an <h3>
  • Use CSS to size headings

Links

Links (or hyperlinks) can be used to connect an HTML page to other pages in the same project or to external sites

Links can connect to files, email addresses, databases--anything with a URL address

<a> Anchor Tag

Links are created with the anchor tag: <a></a> 

<a href="http://www.girldevelopit.com">Girl Develop It</a>

Linking with the <a> tag should not be confused with the <link> tag which is only used in the <head> tag

Creating a Link

Three parts are required to create a link:

  • The <a></a> anchor tag
  • An href attribute: Where the link goes to
  • Content: Text or image that becomes the clickable link 

<a href="http://www.girldevelopit.com">Girl Develop It</a>

Tag Attributes

  • Attributes define additional characteristics of an HTML element or tag
  • An attribute has two parts: Name & Value
<tagname name="value">content</tagname>

href Attribute

href attribute can point to:

  • an external URL
  • an internal page (same root directory or project)
  • a telephone number
  • an email address
  • a section on the same page (a bookmark)

Absolute & Relative Paths

    Absolute Path:
  • Points to the full name or address (including the root directory) of a file or page:
  • "https://girldevelopit.com/community/some-file.html"
    • Relative Path:
  • Points to content from within the same domain or directory:
  • "community/some-file.html"

Link Styles and States

The default appearance of a link is underlined text

The state or status of a link also affects its appearance:

  • An unvisited link is blue
  • A visited link is purple
  • An active link is red

CSS can be used to change the appearance of a link's state

target Attribute

target is an optional anchor tag attribute that tells the browser where to open a link:

_self: In the current window (the default behavior) 

<a href="home.html" target="_self">Link Text</a>

_blank: In a new browser window 

<a href="home.html" target="_blank">Link Text</a>

Images

<img/> Tag

Using the image tag, images can be added to a web page 

<img src="images/gdi-logo.png" alt="GDI Logo"/>

<img/> Tag

The <img/> tag is our first example of a self-closing tag.

The forward slash is optional; though some code editors and linters will add it automatically

<img/> Tag Attributes

The image tag has two default attributes: src and alt

  • src: the location and file name of the image
  • alt: a brief description of the image content
<img src="images/gdi-logo.png" alt="GDI Logo"/>

alt Attribute

The alt attribute is not required for browsers to render an image, but it is considered good practice (and accessibility friendly) to always include it for use cases where viewing an image might not be possible:

  • Screen readers for visually impaired users
  • A broken image link or mis-spelled file name
  • Low bandwidth internet use

Good Practices When Working with Images

  • Save images locally; linking to external sources is not always reliable
  • Properly credit the original image source or owner
  • Work with larger images
    • Compress them to reduce file size
    • Adjust/resize image dimensions with CSS

Image compression site: tinypng.com
