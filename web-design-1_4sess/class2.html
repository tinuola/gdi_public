What is Semantic HTML?

An HTML element whose name/tag describes the type of content it holds:

headings, paragraph, list,

table, form, button,

header, footer, nav,

section, main, article,

video, figure, image, etc

Non-Semantic HTML

HTML elements that do not describe content or have any meaning:

div, span

Semantic vs. Non-Semantic HTML

<div>Watch Season 1</div>
<button>Watch Season 1</button>

Non-Semantic HTML Tags

Two common non-semantic HTML tags:

  • <div>: Used as a generic container for grouping content
  • <span>: Used as a generic wrapper to style inline content

Using a <div> Tag

Grouping content that belongs together

<div>
  <h2>Season 1</h2>

  <h3>Episode 5</h3>

  <p>While on a mission, Lorca is captured by the Klingons 
  and unexpectedly finds himself in the company of prisoner of war

</div>

div: The Content Division element (MDN)

Using <span> Tag

<p>Watch Season 1 and stay tuned for <span>Season 2</span></p>

span: The Content Span element (MDN)

Using divs and spans

<div> and <span> tags have no visible visual effect on content or page layout until CSS is applied to them

Why Use Semantic HTML?

  • Makes our code accessible
  • Improves SEO ranking
  • Easier to read and maintain

*A Quick Intro to Accessibility

*(Accessibility is an important and exhaustive topic; this series can only touch the basic beginnings of it.)

Web Accessibility

Web accessibility is the practice of ensuring our site is usable and navigable, regardless of who is using our sites, where and how they are using our sites

Accessibility is often shortened to A11Y

What is Accessibility? (MDN)

Example Basic Qualities of an Accessible Site

  • Semantically coded HTML
  • Sufficients color contrast between background and text
  • Using legible font styles and sizes
  • Proper use of alt attributes for images
  • Mindful use of animation, or options to diable it

Accessibility as a Mindset

  • Accessibility is a wide and growing topic
  • Accessibility is no longer optional
  • Accessibility is a collective responsibility: Designers, developers, product managers, business owners and organizations are all included in the process of creating an accessible digital product and user experience

Developing an Accessibility Mindset

  • Cultivate an A11Y-first development practice
    • Think accessibility before aesthetics
  • Run sites/projects through A11Y plugins, such as:
    • WAVE - Web Accessibility Evaluation Tool
    • SiteImprove Accessibility Checker
    • axe DevTools
    • W3C Markup Validation Service
  • Educate yourself about accessibility
    • Read articles and tutorials
    • Invest in A11Y courses/training

A Few Good A11Y Resources

  • ALLY-101
  • Accessibility Overview (MDN)
  • Web Accessibility Initiative
  • Deque Courses

Anatomy of CSS

CSS syntax or rule consists of a selector and at least one or more property-value declarations: 

selector {
  property: value;
}

For example:

p {
  color: yellow;
  background-color: black;
}

CSS & HTML

Some HTML elements have default style, but CSS allows for added customization

Without CSS, an HTML page is merely a page with raw, unformatted text, images and links

Applying CSS to HTML

  • Inline CSS
  • Embedded or Internal CSS
  • External CSS Stylesheet
  • JavaScript

CSS Selector

The selector is used to select or find which elements on the HTML page will be given the styles inside the curly braces. 

selector {

  property: value;

}
p {

  color: yellow;

  background-color: black;

}

Three Common CSS Selectors

  • Element or Type selector
  • Class selector
  • ID selector

Element or Type Selector

  • Selects all elements of the same type
  • The CSS rule below targets all p tags or paragraph elements 
p {

  color: yellow;

  background-color: black;

}

Class Selector

Class selectors are prefixed with a "." dot, then followed by the class name 

.product-text {

  color: purple;

}

Class Selector

  • Class selectors are declared in markup, that is, they assigned to HTML elements in the HTML file
  • When we want multiple elements to share a style, we give them the same class name
  • The style rule for the class selector is then declared in CSS 
<p class="product-text">This is some product text.</p>

<p>This is another paragraph of text.</p>

<h4 class="product-text">This is headline text.</h4>

Class Selector

The CSS rule below selects all elements with the class of product-text in the markup and applies the color purple 

.product-text {

  color: purple;

}
<p class="product-text">This is some product text.</p>

<p>This is another paragraph of text.</p>

<h4 class="product-text">This is headline text.</h4>

ID Selector

ID selectors are prefixed with a "#" symbol, then followed by a name 

#product-name {

  background: tomato;

}

ID Selector

Unlike the class selector, the ID selector is associated with only one target element on the HTML page 

<h2 id="product-name">First Product Title</h2>

<h2 id="needs-a-new-id">Second Product Title</h2>

IDs cannot be shared among elements

Applying Multiple Class Names

Elements can have more than one class name or selector applied to them

  color: purple;

Mixing Selectors

An element can have both a class name (or multiple class names) and an ID 

<h2 class="product-text" id="product-name">First Product Title</h2>

Working with Class & ID Selectors

A Few Good Practices
  • Give selectors meaningful names
  • Use single word or hyphenated naming
  • Use class selectors more than IDs
  • Use ID selectors for:
    • Internal bookmarking
    • Elements to be targeted by JavaScript

Sizing Units

Size Units

  • Pixels: px
  • Root element font-size: rem
  • Element font-size: em
  • Percentage: %
  • Viewport height: vh
  • Viewport width: vw

CSS length data types (MDN)

Absolute vs Relative Sizing

Any value set using px (pixels) is an absolute value, meaning the dimension is fixed or static

Most CSS sizing units are relative values, meaning their dimension/behavior is relative to/based off something else, such as a parent element, the default root/html element, viewport height or width, etc

Relative Sizing with the rem Root Element

The default font-size value of the root html element is 16px

Other elements can set their font sizes relative to the root element 

html {
  font-size: 16px;
}

p {
  font-size: 1rem;
}

In the snippet above, the font-size of the p selector is 1rem or 100% of the value of the root html selector, meaning it is also 16px

rem Sizing

When the default value of the root html element is changed, any other element dependent on its value will also be affected 

html {
  font-size: 20px;
}

p {
  font-size: 1rem;
}

img {
  width: 10rem;
}

The font-size of the p selector will now be 20px

The width of the img selector will be 200px

Fonts

Web Safe Fonts

  • Web Safe Fonts are font families that all browsers can display by default
  • We don't have to load font files or link to external font libraries to use them
  • But they are limited in number, and basic
p {
  font-family: "Times New Roman"; /* Specific font name */
}

h2 {
  font-family: serif; /* Generic name */
}

h3 {
  font-family: "Arial", sans-serif; /* Comma-separated list */
}

Custom Fonts

Custom fonts add visual interest

They can establish brand recognition

Depending on how they are designed or used, they can elevate (or hurt!) the aesthetics of a site

Using Custom Fonts

To apply custom fonts, we either add font files directly into our project directory, or we can link to external font libraries (free or premium) such as Google Fonts, Adobe Fonts, myFonts, etc.

Linking to External Fonts

  • We use the link tag to connect to external font urls
  • The link is added to the head tag, and ideally should be placed before a site or project's default stylesheet 
<!DOCTYPE html>
  <html>
    <head>
    ...
      
      
      
      
      
      
      <link href="style.css" rel="stylesheet" type="text/css" />
      <title>Title of the page </title>
    </head>
    ...
    ...
  </html>
