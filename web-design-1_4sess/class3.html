Box Model Properties

  • Content: Text, image, form, etc.

  • Border:The wrapper around the content and its padding

padding Property

The padding property controls the amount of clear space around an element

CSS Padding Property (MDN)

padding Property Syntax

<p class="box">Hello World</p>

<p class="box">Goodbye World</p>

.box {

  padding: 10px;

  background-color: yellowgreen; /* For visuals */

}

padding Property Syntax

.box {

  padding-top: 10px;

  padding-right: 5px;

  padding-bottom: 10px;

  padding-left: 5px;

}
padding Property Shorthand Syntax
.box {
  padding-top: 10px;
  padding-right: 5px;
  padding-bottom: 10px;
  padding-left: 5px;
}

Is the same as:

.box {
 padding: 10px 5px 10px 5px; /* top, right, bottom, left */
}

.box {
 padding: 10px 5px;  /* top & bottom; left & right */
}

When all sides have the same value:

.box {
 padding: 10px; 
}

Display Flow

Display Flow

Every element has a default flow--the way that it renders on the page before any CSS styling is applied

An element is either:

  • Stacked on top of another element
  • Positioned side by side with another element

Block and Inline Elements

  • Elements that stack are on top of other elements are known as Block level elements
  • Elements that are positioned side-by-side on the same line are Inline elements

Block vs Inline Behavior

  • Block elements start a new line or live on their own line; they take up the full width of their line or their parent container
  • Inline elements don't start a new line and take up only as much space as their content needs

Block Elements

Block-level elements include:

  • Headings (h1, h2, etc)
  • Paragraphs
  • Divs
  • Lists

Block Level Elements (MDN)

Inline Elements

Inline-level elements include:

  • Anchor "a" tags
  • Images
  • Spans

Inline Level Elements (MDN)

display Property

The CSS display Property

  • CSS can be used to change an element's default flow or display property
  • A block element can be changed to an inline element and vice versa

display Property Syntax

Any paragraph element will now be an inline element

p {

display: inline;

}

Any span element will now be a block element

span {

  display: block;

}

Inline Element Caveat

Setting margin-top or margin-bottom to inline elements (either default, or converted block elements) has no layout effect 

<span>Hello</span>

<p>Goodbye</p>

span, p {
  background-color: red;
}

p {
  display: inline;
  margin-top: 16px;
}

span {
  margin-bottom: 16px;
}

display: inline-block

When we want an element to be inline and have block-like behavior (the ability to have a margin-top or margin-bottom) we can set its display value to inline-block

display: inline-block

<span>Hello</span>

<p>Goodbye</p>

span, p {
  background-color: red;
}

p {
  display: inline-block;
  margin-top: 16px;
}

span {
  display: inline-block;
  margin-bottom: 16px;
}

Layout Gotchas

Some default properties of CSS can lead to unexpected layout behavior

Three common culprits are:

  1. The box-sizing property
  2. Browser defaults
  3. Margin collapsing

box-sizing Property

The CSS box-sizing property determines how we calculate the total width and height of an element

It has two values:

  1. content-box
  2. border-box
/* Default value of all elements */

box-sizing: content-box;

/* border-box: your layout's best friend! */

box-sizing: border-box;

box-sizing Demo

Let's Develop It!

Copy the code below to set the box-sizing property of all elements in your project to border-box 

/* This declaration will target every element in the project */

* {

  box-sizing: border-box;

}

Browser Defaults

  • Most HTML elements have one or more default browser values, that is, they have some basic CSS style that influences their appearance on a page
  • Some of these defaults can throw off our page layout (example: box-sizing: context-box)
  • Others may not be needed or wanted

Browser Defaults

Examples of browser defaults:

  • A p tag has a margin-top and margin-bottom of 16px
  • A ul-li tag pair has a dark, circular marker or bulletpoint
  • An a tag is underlined and has colors for different states
  • An h1 tag has a font-size of 32px

Resetting CSS

  • "CSS Reset" is one way to globally remove some of these defaults in our project
  • With a "CSS Reset", we are writing new CSS values to overwrite the browser defaults

CSS Resets

You can create your own reset rules or use well-known reset stylesheets such as:

  • Andy Bell's CSS Reset
  • Collection of CSS Resets (Article)
  • Normalize.css

CSS Reset Demo

Two Ways to Add CSS Resets

  • Create a reset.css stylesheet file for your project
    1. Add your CSS overwrites to the new stylesheet
    2. Link the stylesheet in the head element/section of your HTML file
    3. Make sure the link comes before the project's stylesheet link
  • OR
  • Add the CSS overwrites or reset values at the start/ top of your project's stylesheet

background-image Property

Working with Images in CSS

Inline Images

  • Inline images are part of the content (that is, they are part of the HTML markup)
  • They are added with the <img> tag

Background Images

  • Background images are part of the CSS code
  • They are added with the CSS background-image property
  • The background-image property has a url() value for the location of the image--either to an internal path or to an external link 
div {

  background-image: url('wallpaper.png');

}

background-image (MDN)

Use an Inline or Background Image?

  • Inline images are semantic--they are relevant to the page's content and are better for accessibility
  • Background images are primarily decorative and non-semantic, that is, they don't add meaning to the content

Styling CSS Images

Both inline and background images can be styled using CSS properties such as:

  • Width
  • Height
  • Border
  • Opacity
  • and more!

width, height, border, opacity (MDN)

Background Image Properties

Background images can be controlled using additional CSS properties such as: 

div {

  background-image: url('wallpaper.png');

  background-repeat: no-repeat;

  background-size: cover;

  background-position: center center;

  width: 400px; /* Not a background style property */

  height: 400px; /* Not a background style property */
  
}

CSS Background style properties(MDN)

Background Image Sizing

The dimension of a background image is determined by the size or amount of its content container or by specific CSS sizing properties such as width, max-width, height, min-height, etc.
