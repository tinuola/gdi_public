Box Model Properties

  • Content: Text, image, form, etc.
  • Padding:The space around the content, or the distance between the content and its borders
  • Border:The wrapper around the content and its padding
  • Margin: The distance between an element and its neighbors

Padding Property

The padding property controls the amount of clear space around an element

CSS Padding Property (MDN)

Padding Property Syntax

<p class="box">Hello World</p>

<p class="box">Goodbye World</p>

.box {

  background-color: yellowgreen; /* For visual demo */

  padding: 10px;

}

Padding Property Syntax

.box {

  padding-top: 10px;

  padding-right: 5px;

  padding-bottom: 10px;

  padding-left: 5px;
}
Padding Property Shorthand Syntax
.box {
  padding-top: 10px;
  padding-right: 5px;
  padding-bottom: 10px;
  padding-left: 5px;
}

Is the same as:

.box {
 padding: 10px 5px 10px 5px; /* top, right, bottom, left */
}

.box {
 padding: 10px 5px;  /* top & bottom; left & right */
}

When all sides have the same value:

.box {
 padding: 10px; 
}

Border Property

The border property demarcates the area or creates a visible boundary around a content and its padding (inner space)

CSS Border Property (MDN)

Border Property Syntax

<p class="box">Hello World</p>

<p class="box">Goodbye World</p>

.box {
  
  background-color: yellowgreen;
  
  padding: 10px;

  margin: 16px;
  
  border: 3px solid black; 
}

Border Property Syntax

The border property is a shorthand for: the width, style, and color of a border; these properties can be declared individually 

.box {
  border: 3px solid black; 
}

... is the same as:

.box {

  border-width: 3px;

  border-style: solid;

  border-color: black;  

}

Display Flow

Every element has a default flow--the way that it renders on the page before any CSS styling is applied. It is either:

  • Stacking on top of another element
  • Side by side with another element

Block and Inline Elements

  • Elements that stack are: Block elements
  • Elements that are side-by-side on the same line are Inline elements

This is also referred to as an element's display property.

Block vs Inline Behavior

  • Block elements start a new line or live on their own line; they take up the full width of their line or their parent container
  • Inline elements don't start a new line and take up only as much space as their content needs

Block Elements

Block-level elements include:

  • Headings (h1, h2, etc)
  • Paragraphs
  • Divs
  • Lists

Block Level Elements (MDN)

Inline Elements

Inline-level elements include:

  • Anchor "a" tags
  • Images
  • Spans

Inline Level Elements (MDN)

The CSS "Display" Property

  • CSS can be used to change an element's default flow or display property
  • A block element can be changed to an inline element and vice versa

The CSS "Display" Property

/* This paragraph element will now be an inline element */

p {

display: inline;

}

/* This span element will now be a block element */

span {

  display: block;

}

Layout Gotchas

Some default properties of CSS can lead to unexpected layout behavior. Three common culprits:

  1. The box-sizing property
  2. Browser defaults
  3. Margin collapsing

box-sizing Property

The CSS box-sizing property determines how we calculate the total width and height of an element. It has two values: 

/* content-box is the default box-sizing 
   value of all elements */
box-sizing: content-box;

/* border-box will be your layout's best friend! */
box-sizing: border-box;

box-sizing Demo

Layout Tip

Set the box-sizing property of all elements in your project to border-box 

/* This declaration will target every element in the project */

* {
  box-sizing: border-box;
}

Browser Defaults

  • Most HTML elements have one or more default browser values, that is, they have some basic CSS that influences their appearance on a page
  • Some of these defaults can throw off our page layout (ex. box-sizing); others may not be needed or wanted

Browser Defaults

Examples of browser defaults:

  • A p tag has a margin-top and margin-bottom of 1em
  • A ul-li tag has a dark, circular marker or bulletpoint
  • An a tag is underlined and has colors for different states
  • An h1 tag has a font-size of 32px

Resetting CSS

  • "CSS Reset" is one way to globally remove some of these defaults in our project
  • With a "CSS Reset", we are writing new CSS values to overwrite the browser defaults

CSS Resets

You can create your own reset rules or use any number of well-known reset stylesheets such as:

  • Andy Bell's CSS Reset
  • Collection of CSS Resets (Article)
  • Normalize.css

CSS Reset Demo

Two Ways to Reset

  1. Add a separate reset.css stylesheet to your project with the link tag in the head section of your HTML page
    • Make sure the link comes before the project's stylesheet link
  2. Include the new CSS rules at the top of your project's stylesheet

Images

Inline Images

  • Inline images are part of the content (i.e., the HTML markup)
  • They are added with the img tag

Background Images

  • Background images are part of the CSS
  • They are added with the CSS background-image property 

div {
  background-image: url('wallpaper.png');
}

Background Image Properties

Background images can be controlled using additional CSS properties such as: 

div {
  background-image: url('wallpaper.png');
  background-repeat: no-repeat;
  background-size: cover; /* cover, contain, or absolute size */
  background-position: center center;
  width: 400px;
  height: 400px;
}

Styling CSS Images

Both inline and background images can be styled using CSS properties such as:

  • width
  • height
  • border
  • opacity

Use an Inline or Background Image?

  • Inline images are semantic--they are relevant to the page's content and are better for accessibility
  • Background images are primarily decorative, that is, they don't add meaning to the content

Tip: Responsive Inline Images

The following CSS declaration is often globally applied to the img tag to make inline images easier to work with: 

img {
  max-width: 100%;
  display: block;
}
