Responsive Web Design

Responsive Web Design

A design and development approach to creating web pages, web content and web applications that respond or adapt to different devices and screen sizes

Responsiveness in Web Browsers

We can check the responsiveness of our websites and applications in at least three ways:

  • Manually resizing our browser windows
  • Physically verifying responsiveness on actual devices
  • Using the Responsive Viewport Mode in our browsers' Developer Tools (aka Dev Tools)

Responsive Viewport Mode

  • Open the Dev Tools Inspector with the keyboard shortcut:
    • cmd opt c (Mac) or ctrl opt c (PC)
  • Toggle the device icon in the inspector window

Fluid / Responsive Images

A fluid image does not grow beyond the width of its parent container, and can scale down as its parent container shrinks

The following CSS declaration is often globally applied to the img tag to make inline images fluid: 

img {

  max-width: 100%;

  display: block;

}

Responsive Image Tip

Always wrap an image element inside a parent container, typically a set of div tags; then set a width on the parent container 


  



div {
  width: 400px; 
}

img {
  max-width: 100%;
  display: block;
}

Let's Develop It!

  • Add the responsive image code snippet to your main stylesheet or to your reset.css stylesheet 
img {

  max-width: 100%;

  display: block;

}

Responsive Web Design with CSS

Responsive Web Design with CSS

We can make our pages responsive using three features in CSS:

  • Media Queries
  • *CSS Flexbox
  • *CSS Grid

*CSS Flexbox and CSS Grid are topics in other GDI Frontend courses: Web Design 2 and CSS Grid Basics

Media Queries

  • Media queries tell the browser at what breakpoint or screen size to apply a specific rule or set of rules
  • Media queries are applied in the CSS stylesheet
@media screen and (min-width: 600px){

  h2 {
    font-size: 30px;
  }

  p {
    font-size: 14px;
  }

}

A Beginner's Guide to Media Queries (MDN)

Basic Media Query Syntax

A media query contains:

  • The @media keyword
  • Optionally, a media type followed by 'and'
  • When to apply the media query
  • The selector(s) to target and the CSS rule(s) to run
@media screen and (min-width: 600px){

  h2 {
    font-size: 30px;
  }

  p {
    font-size: 14px;
  }

}

Using Media Queries (MDN)

Basic Media Query Syntax

The media query rule below means:

When the screen width is at 600px or higher, h2 elements should be 30px in size, and p elements should be 14px 

@media screen and (min-width: 600px){

  h2 {
    font-size: 30px;
  }

  p {
    font-size: 14px;
  }
  
}

Media Query Breakpoints

A breakpoint is the screen size / resolution / width at which a layout should change 

@media screen and (min-width: 600px){...}


@media (min-width: 800px) and (max-width: 1000px) { ... }


@media screen and (max-width: 1200px){ ... }


@media (min-width: 30em) and (orientation: landscape) { ... }

Non-metric conditions such as orientation, hover, prefers-reduced-motion, and others, can also be used in media queries, but the most common is screen size

Standard Breakpoints

A common way to determine or set breakpoints is by device size: 

/* Default: Extra-small devices such as small phones 
(less than 640px) */


/* Small devices such as large phones (640px and up) */
@media only screen and (min-width: 640px) {...}


/* Medium devices such as tablets (768px and up) */
@media only screen and (min-width: 768px) {...}


/* Large devices such as laptops (1024px and up) */
@media only screen and (min-width: 1024px) {...}


/* Largest devices such as desktops (1280px and up) */
@media only screen and (min-width: 1280px) {...}

min-width vs max-width

Understanding when or how to use the min-width and the max-width properties in a media query can initially be confusing!

The choice of which to use may come down to preference: what you prefer the default approach of your CSS structure to be

Mobile-First: min-width

If you prefer to start your CSS by writing the mobile state or styles first, and then size up with media queries, use min-width 

/* Default - Mobile State */

.box {
  background-color: tomato;
}

@media screen and (min-width: 700px){
  .box {
    background-color: goldenrod;
  }
}

@media (min-width: 1000px){
  .box {
    background-color: mediumaquamarine;
  }
}

Desktop-First: max-width

If you prefer to start your CSS by writing the desktop state or styles first, and then size down with media queries, use max-width 

/* Default - Desktop State */

.box {
  background-color: mediumaquamarine;
}

@media screen and (max-width: 999px){
  .box {
    background-color: goldenrod;
  }
}

@media (max-width: 699px){
  .box {
    background-color: tomato;
  }
}

Custom Breakpoints

You can and should set breakpoints according to your project's design and requirements 

@media only screen and (min-width: 700px) {...}

@media only screen and (max-width: 924px) {...}

@media only screen and (hover) {...}

Your project may need only a handful of breakpoints or many!

Organizing Media Queries

Two common approaches for organizing media queries

By Selector

All media queries associated with a particular selector are placed immediately after the default declaration of the selector 

<div class="box">Hello World</p>

/* Default */
.box {
  height: 150px;
  width: 150px;
  border: 2px solid black;
  background-color: tomato;
}

@media only screen and (min-width: 640px) {
  .box {
    background-color: goldenrod;
  }
}

@media only screen and (min-width: 768px) {
  .box {
    background-color: magenta;
  }
}

@media only screen and (min-width: 1024px) {
  .box {
    background-color: mediumaquamarine;
  }
}

By Breakpoint

Selectors that are changing at a shared breakpoint are grouped together under that breakpoint 

<div class="box">Hello World</p>

<p>Lorem ipsum dolor sit amet.</p>

.box {
  height: 150px;
  width: 150px;
  border: 2px solid black;
  background-color: tomato;
}

p {
  font-size: 14px;
}

/* Other default code declarations ... */

/* List all media queries after default rules */

@media only screen and (min-width: 768px) {
  .box {
    background-color: magenta;
  }

  p {
    font-size: 30px;
  }
}

Order Matters

Because of CSS's cascading algorithm, order matters when listing media queries

Media queries should be listed in breakpoint order

Media Query Order

<div class="box">Hello World</p>

/* Default */
.box {
  height: 150px;
  width: 150px;
  border: 2px solid black;
  background-color: tomato;
}

/* Out of order */
@media only screen and (min-width: 768px) {
  .box {
    background-color: magenta;
  }
}

/* Out of order */
@media only screen and (min-width: 640px) {
  .box {
    background-color: goldenrod;
  }
}

@media only screen and (min-width: 1024px) {
  .box {
    background-color: mediumaquamarine;
  }
}

The background-color: magenta will not be applied

Part 2: Tablet breakpoint styles

  2. Set the container's overall padding to 24px
  3. Give the page title a font-size of 32px

Part 3: Mobile breakpoint styles

  1. Reduce the container's padding to 16px
  2. Reduce the page title's font-size to 24px
  3. Give the callout-container an overall padding and margin-top of 16px
  4. Reduce the font-size of the callout-heading to 18px and that of the recipe headings to 20px

Let's Develop It!

Part 4: Centering the Container

    In the default / desktop CSS state:
  1. Give the main container, a margin-left and margin-right value of auto
  2. Set the margin-top to 80px
  3. Apply a border-radius of 16px
  4. In the markup, give the Omelette image a class name of recipe-img
  5. In the stylesheet, apply the same border-radius value to the new selector, and also to the callout-container selector

Let's Develop It!

Part 5: Styling the Nutrition Table

    To the data-row selector:
  1. Apply a border-bottom of 1px solid #5f564d
  2. Apply padding-bottom and margin-top of 8px
  3. Apply a padding-left of 30px to the dt selector
  4. Set the font-weight of the dd selector to 600, and give it a color of #f3e5d7

Let's Develop It!

Part 6: Styling the Nutrition Table (contd.)

    The content in the table needs more spacing; typically, we should fix this with Flexbox, but we haven't learned that yet! Here's an alternate approach:
  1. In your markup, give each of the four dd selectors its own class name, for example:
    • dd-1, dd-2, dd-3, dd-4
  2. In your stylesheet, apply the following margin-left values to each respective dd class name you've just created:
    • 200px, 213px, 202px, 228px

Let's Develop It!

Part 7: Styling the Footer

    The footer would be better outside of the container, not inside of it:
  1. In your markup, move the footer section out of the container; place it before the closing body tag
  2. Give the p tag in the footer a class name of attribution
  3. Apply a top and bottom padding of 30px to the footer selector
  4. Apply text-align: center to the footer selector
  5. Apply a display property of inline, and a padding of 5px to the attribution selector
